Mobile Editor Layout Strategy

Your desktop editor has 3 panels (left sidebar | 3D viewport | right sidebar). How should this transform on mobile?

← DEXO ↩ ↪ ⋮
3D VIEWPORT
(full screen)
📚 Library ✋ Select ✏️ Edit ⚙ Props
↕ Bottom sheet slides up with Library / Properties

A: Bottom Sheet Drawers

Full-screen viewport + bottom action bar. Library and Properties open as draggable bottom sheets (half → full height). Your prompt's recommended approach.

← DEXO ↩ ↪ ⋮
Viewport Library Properties
3D VIEWPORT
(swipe to switch tabs)
✋ Select ✏️ Edit 🤖 AI

B: Tab-Based Navigation

Swipeable tabs: Viewport | Library | Properties. Always visible tab bar at top. Simpler but you can't see viewport while adjusting properties.

← DEXO ↩ ↪ ⋮
3D VIEWPORT (60%)
Properties panel (40%)
swipe left/right: Library ↔ Properties
✋ Select ✏️ Edit 🤖 AI

C: Split View (60/40)

Viewport on top (60%), swipeable Library/Properties panel below (40%). Always visible, like iPad split view. Good for tablets, cramped on phones.

Key Trade-offs

Bottom Sheets (A) Win When

  • Phone screens are small (390px)
  • 3D viewport needs max space
  • Users interact with one panel at a time
  • Already have vaul drawer library

Tabs/Split (B/C) Win When

  • Users switch panels frequently
  • Need to see properties while viewing 3D
  • Tablet is the primary mobile device
  • Simpler state management
